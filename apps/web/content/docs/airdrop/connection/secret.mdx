## Overview

The Secret Connection airdrop provides a secure authentication method using secret keys. This configuration is ideal for server-to-server communication where you need to authenticate using a shared secret.

## Configuration Options

### Authentication

The secret connection requires the following configuration:

- **secret_key**: A secure secret key for authentication (minimum 32 characters)
- **endpoint_url**: The base URL for the service
- **timeout**: Connection timeout in milliseconds (default: 30000)

### Environment Variables

Set the following environment variables:

```bash
SECRET_KEY=your_secret_key_here
ENDPOINT_URL=https://api.example.com
```

### Endpoints

The configuration includes the following endpoints:

- **auth**: `/auth/secret` - Authentication endpoint
- **verify**: `/auth/verify` - Token verification endpoint
- **refresh**: `/auth/refresh` - Token refresh endpoint

## Usage

After installation, the manifest file will be available at:

```bash
~/manifest.yml
```

### Example Configuration

```yaml
version: "2"
name: "Secret Connection"
description: "Airdrop configuration for secret-based authentication connections"
type: "airdrop-connection"
auth_type: "secret"

connection:
  authentication:
    type: "secret"
    secret_key: "${SECRET_KEY}"
    validation:
      required: true
      min_length: 32
```

## Error Handling

The configuration includes robust error handling with:

- Automatic retry on failure (max 3 attempts)
- Exponential backoff strategy
- Retry on specific HTTP status codes (408, 429, 5xx)
- Configurable timeout and retry delays

## Security Notes

- Keep your secret key secure and never commit it to version control
- Use environment variables for sensitive configuration
- Ensure the secret key meets the minimum length requirement (32 characters)
- Consider rotating your secret keys regularly
